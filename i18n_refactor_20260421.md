# i18n 全局耦合重构总结

**日期**: 2026-04-21
**方案**: 方案1 - 参数传递（松耦合）
**状态**: ✅ 核心重构已完成

---

## 📋 重构概述

### 问题
- `main.js` 通过 `window.i18n` 获取 i18n 实例
- 造成全局命名空间污染
- 与 i18n 实现紧耦合，不利于模块化和测试

### 解决方案
采用**参数传递**模式，实现松耦合：
- i18n.js 导出实例
- main.js 通过 ES6 导入获取实例
- 移除对 `window.i18n` 的直接依赖

---

## ✅ 已完成的更改

### 1. 修改 `i18n/i18n.js`

**位置**: `public/js/i18n/i18n.js:448-451`

**变更前**:
```javascript
// 创建全局实例
window.i18n = new I18n();
```

**变更后**:
```javascript
// 创建并导出 i18n 实例
export const i18nInstance = new I18n();

// 为了向后兼容，暂时保留全局实例
window.i18n = i18nInstance;
```

**内部优化**:
- 将所有 `window.i18n` 引用改为 `i18nInstance`
- 移除模块内部对全局变量的依赖

---

### 2. 修改 `main.js`

**位置**:
- `public/js/main.js:33` - 新增导入
- `public/js/main.js:55` - 移除 window 读取

**变更前**:
```javascript
// 导入部分
import { EVENTS } from './config/events.js';

// init 函数
async function init() {
  try {
    // 设置全局i18n引用（在初始化前就设置）
    i18n = window.i18n;
```

**变更后**:
```javascript
// 导入部分
import { EVENTS } from './config/events.js';
import { i18nInstance } from './i18n/i18n.js';

// init 函数
async function init() {
  try {
    // 使用导入的 i18n 实例（不再依赖全局变量）
    i18n = i18nInstance;
```

**优势**:
- ✅ 显式依赖，易于理解
- ✅ 便于测试（可 mock i18nInstance）
- ✅ 不依赖全局变量

---

### 3. 修改 `utils/i18n-helper.js`

**位置**: `public/js/utils/i18n-helper.js:1-35`

**变更前**:
```javascript
/**
 * 获取全局 i18n 实例
 * 在 index.html 的脚本中，i18n.js 将 window.i18n 设置为全局 I18n 实例
 */
export function getI18n() {
  if (!window.i18n) {
    throw new Error('i18n 系统未初始化。请确保在 init() 之前已加载 i18n.js');
  }
  return window.i18n;
}

export function isI18nReady() {
  return typeof window.i18n !== 'undefined' && window.i18n !== null;
}
```

**变更后**:
```javascript
/**
 * i18n 辅助模块 - 统一访问 i18n 实例
 * 使用导入的实例而非全局变量，实现松耦合
 */
import { i18nInstance } from '../i18n/i18n.js';

/**
 * 获取 i18n 实例
 * 从 i18n.js 导入的实例获取，避免依赖全局变量
 */
export function getI18n() {
  if (!i18nInstance) {
    throw new Error('i18n 系统未初始化。请确保在 init() 之前已加载 i18n.js');
  }
  return i18nInstance;
}

export function isI18nReady() {
  return typeof i18nInstance !== 'undefined' && i18nInstance !== null;
}
```

**优势**:
- ✅ 不再依赖 `window.i18n`
- ✅ 使用模块级导入
- ✅ 类型安全（可添加 TypeScript 类型）
